USB Info Stealer
Operational Deployment Guide

A complete, tested methodology for deploying a USB-based information stealer against Windows 10/11 targets — from environment setup to post‑execution.

# Overview

This document provides a complete, tested methodology for deploying a USB-based information stealer against Windows 10/11 targets. The payload collects system metadata, browser credentials, Windows Vault entries, and select user files — exfiltrating locally to the USB or remotely to a configured server.

Status: Production-ready. Tested on Windows 11 build 26200.

# 1. Environment Setup

Build Machine Requirements

  • Windows 10/11 (x64)
  • Python 3.8+ (3.14.5 used in testing)
  • Administrator privileges (optional, recommended for full testing)
  • USB drive (≥4GB, FAT32 formatted)

Dependency Installation

# Install required Python packages pip install pyinstaller selenium requests pywin32 netifaces

Note: pyusbhid is deprecated and not used in this build.

# 6. Deployment Vectors

  • Vector A: Autorun (Legacy Systems) — Windows 7/8 with autorun enabled. Insert USB → payload executes automatically.
  • Vector B: Manual Execution — Windows 10/11 (autorun disabled by default). User must double-click usb_stealer.exe.
  • Vector C: Social Engineering — Rename executable to Invoice_2026.pdf.exe, README.txt.exe, or Confidential_Docs.exe. Set icon to PDF or folder icon using resource editors.

# 8. Operational Notes

VectorMitigation
Windows DefenderCode signing, UPX packing, obfuscation
Network IDSUse local exfil only, or encrypt traffic
User behaviorKeep execution silent, minimize visible windows

AV Evasion Techniques

  1. Pack the binary: upx --best usb_stealer.exe — UPX download
  2. Obfuscate strings: Use base64 encoding for sensitive constants
  3. Delay execution: Add time.sleep(30) at start to bypass sandboxes
  4. Avoid registry writes: Keep payload memory-resident where possible

Recommended Modifications

  • Replace SERVER_URL with your own C2 endpoint
  • Modify USER_DIR targets for specific user folders
  • Adjust file extension list based on intelligence
  • Implement persistent installation if required
